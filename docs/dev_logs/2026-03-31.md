### 4. 启动链路问题已定位并修正

本窗口中间一度出现的真实问题：

1. `Mindhikers CMS Core` 在 must-use 插件页可见
2. 但 `mh_homepage` 不存在
3. 访问 `edit.php?post_type=mh_homepage` 会报“无效的文章类型”
4. `wp-json/mindhikers/v1/homepage/zh` 返回 `rest_no_route`

根因判断：

1. 原先把 `Mindhikers_Cms_Core::boot()` 放在 `bootstrap.php` 文件末尾
2. 该启动方式在当前 mu-plugin 场景下表现不稳定，出现了“类和文件被加载，但 hook / 注册链路没真正生效”的怪状态

本次修正：

1. 改为在真正的 mu-plugin 入口文件 `mindhikers-cms-core.php` 中显式调用：
   - `Mindhikers_Cms_Core::boot()`
2. 同时在 `boot()` 中加入当前请求的核心状态注册：
   - `registerPostType()`
   - `registerMeta()`

修正后结果：

1. `mh_homepage` 已恢复注册
2. 后台菜单中已出现：
   - `Mindhikers Homepages`
   - `Mindhikers CMS`
3. `https://wordpress-l1ta-production.up.railway.app/wp-json/mindhikers/v1/homepage/zh`
   已返回结构化 JSON

## 协作经验

本窗口新增了一条重要协作规则，后续应写入 `docs/rules.md`：

1. 当遇到类似浏览器、平台、远端交互卡点时，AI 自己最多尝试 3 次
